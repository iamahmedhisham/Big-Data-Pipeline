Overview

This project implements a Big Data Pipeline designed to handle streaming data with high throughput and scalability. It simulates real-world use cases such as processing social media feeds or IoT sensor data in real-time.

The pipeline ingests data, processes it using distributed frameworks, and stores results for analysis, leveraging containerization for easy deployment.

Architecture

The pipeline follows a modular architecture with distinct components for data ingestion, processing, and storage:

  • Data Source: Simulates streaming data input.
  • Kafka: Acts as the message broker for data ingestion.
  • Spark: Processes data in real-time using streaming capabilities.
  • Cassandra: Stores processed data for querying and analytics.

Diagram illustrating the flow of data through the pipeline.

Technology Stack

  • Apache Kafka: Distributed streaming platform for data ingestion.
  • Apache Spark: Unified analytics engine for big data processing.
  • Apache Cassandra: Distributed NoSQL database for storage.
  • Docker: Containerization for deployment and scalability.
  • Python: Used for scripting and data simulation.

Setup Instructions

Follow these steps to set up the pipeline on your local machine:

  1. Clone the repository: git clone https://github.com/iamahmedhisham/Big-Data-Pipeline.git
  2. Ensure Docker and Docker Compose are installed.
  3. Navigate to the project directory and run: docker-compose up
  4. Verify that Kafka, Spark, and Cassandra containers are running.

For detailed setup, refer to the GitHub repository.

Usage

To simulate data streaming and processing:

# Start the data producer (example command)
python producer.py

# Run Spark job for processing (example command)
spark-submit spark_processing.py

Processed data will be stored in Cassandra. Query results using CQL shell:

docker exec -it cassandra cqlsh
